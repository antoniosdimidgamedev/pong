## 👋 Hey, I'm Antonios Dimi

I build things that run in a terminal.

### 🔭 What I'm working on

**Pong** — a fully self-contained terminal Pong game in a single Python file. No dependencies beyond Python 3.6+ and curses. It started as a weekend toy and grew into: local PvP, Player vs CPU with three difficulty levels (the AI tracks ball direction with configurable jitter), LAN multiplayer with room-based matchmaking and UDP broadcast server discovery, in-game chat with server-side profanity filtering, a web dashboard served on a configurable port (pure stdlib HTTP server + live JSON), a save/load system for CPU games, a server management screen where the host can kick players and join locally, and an easter egg that rickrolls you if you idle too long on the menu. All in ~1800 lines, one file.

I care about software that just works — no npm install, no pip install, no containers. Download it and run it.

### 🧰 Tech stack

Python (curses, sockets, threading, http.server), C, Go, Rust, shell scripting, Linux systems programming, terminal UI design, game networking (custom JSON protocol over TCP), embedded systems.

### 🎮 Interests

Terminal UI / TUI design, game development (especially retro/constrained), LAN multiplayer, zero-dependency software, systems programming, building things that survive `rm -rf node_modules`.
